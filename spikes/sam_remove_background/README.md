# Spike: sam_remove_background

Question tested
- Can a click-to-mask → transparent-object pipeline produce reliable `final_output_file` assets with minimal manual correction?

Expected input
- Representative images with clearly selectable objects.

Expected output
- Mask PNGs and transparent alpha PNGs (final objects) generated by the pipeline; screenshots of the click-to-mask workflow.

Success criteria
- Click/prompt produced masks can be refined and applied to produce clean `final_output_file` images that no longer include neighboring fragments.

Dependency risk
- High: most implementations rely on SAM (PyTorch) and model checkpoints. Treat this as a high-risk spike; prefer conceptual evaluation or mocked backends initially.

How output maps back to VikVec
- `mask_file`: mask PNG produced by the pipeline
- `final_output_file`: final alpha PNG of the object
- `extraction_method`: e.g., `sam_point` or `sam_box`

Notes
- Source example: `MrSyee/SAM-remove-background` demonstrates the pattern; record the repo in `notes.md` if referenced. Do not clone full heavy repos into `vikvec/`.
